## 📖 About The Project

This repository documents my Point of Sale (POS) Application project. It covers all the essential features of a modern retail system, including inventory management, multiple user roles, a robust POS terminal, and insightful dashboards. It features a responsive, beautifully designed user interface reflecting modern aesthetics.

**Key Features:**

- **Role-Based Access Control (RBAC):** Distinct roles (Owner, Manager, Cashier) with restricted permissions and capabilities.
- **POS Terminal:** A streamlined, interactive interface designed for fast and efficient sales processing.
- **Inventory Management:** Fully featured tools for managing products and keeping track of stock levels.
- **User Authentication:** Secure login system, including user profile management with customized avatar uploads.
- **Clean, responsive code:** Built prioritizing modern frontend UI design principles.

### 🛠️ Built With

- [React](https://reactjs.org/)
- [Vite](https://vitejs.dev/)
- [PHP](https://www.php.net/)
- [MySQL (via XAMPP)](https://www.apachefriends.org/)

## 🚀 Getting Started

### 📋 Prerequisites

- Node.js & npm
- XAMPP (for PHP/MySQL support)

### ⚙️ Installation

1. Clone the repository:

```bash
git clone https://github.com/reymartrubio24-beep/pos-app.git
```

2. Navigate to the project folder and install dependencies:

```bash
cd pos-app
npm install
```

3. Setup Database (XAMPP):

Move the `php` API folder to your XAMPP `htdocs` directory (or map the app correctly). Run the provided database initialization/migration scripts.

4. Start the development server for the frontend:

```bash
npm run dev
```

## 💡 Usage

Organized by feature access level:

### 🎯 Application Roles

**Cashiers**

- Access to the real-time POS terminal.
- Ability to log transactions securely and manage the current cart.

**Managers**

- Ability to monitor current inventory.
- Control over basic store configurations and day-to-day metrics.

**Owners**

- Access to absolute system settings.
- Supervise total sales analytics and configure access limits for different user roles.
